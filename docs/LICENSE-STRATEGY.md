# Лицензионная стратегия PulsarOS

## TL;DR

**Наш код** → MIT (свободно используйте)
**Чужой код** → их лицензии остаются

## Компоненты

| Компонент | Лицензия | Что можно |
|-----------|----------|-----------|
| PulsarOS core | MIT | Всё |
| AOSP | Apache 2.0 | Использовать с атрибуцией |
| Linux kernel | GPL-2.0 | Обязаны раскрыть модификации |
| Mali blob | Proprietary | Только бинарник |
| RetroArch / ES-DE / Kodi | GPL-3.0 | Раскрыть модификации |
| Console Launcher / TV Bro | MIT | Свободно |

## Практика

При распространении бинарного образа:
1. Приложить `LICENSE` и `NOTICE`
2. Предоставить исходники модифицированных GPL-компонентов
3. Ссылки на форки → `docs/SOURCES.md`

## Рекомендация

В release-образах **не включать** GPL-приложения по умолчанию.
Предлагать установку через OOBE. Это сохраняет чистоту ядра MIT.
